Events automatically tracked

  • page_view — every visited page (default)
  • scroll, click, file_download, video_* — via GA4 Enhanced Measurement (default)
  • view_item — when a visitor opens a package detail page
  • search — when the homepage search bar is used
  • begin_checkout — when a visitor starts the Razorpay checkout
  • purchase — successful payment, with revenue (₹) attached
  • payment_failed — a Razorpay payment failure

Site Settings

Changes here are saved to Firebase Firestore and apply to every visitor immediately.

Online Payments (Razorpay)

When off, the "Pay Now" button on every package and the booking flow is replaced with a "Call to Book" CTA. Existing bookings are unaffected.

Leave blank to use the default message.

Booking Advance / Commission Rate

The percentage of total trip cost charged via Razorpay to confirm a booking. The remaining balance is collected at the end of the trip. Default is 5%. Each customer can have their own override (set in the Customers tab) which takes priority over this global value.

%

Developer Console Lock

When on, normal visitors cannot open DevTools (F12, Ctrl+Shift+I), right-click for "View Source", or save the page. Admins are always exempt — you keep full DevTools access whenever you're logged in. Toggle off to allow everyone to use the console (useful when debugging on the live site).

Database Backup & Mirror

Mirror all Firestore data (packages, users, bookings, gallery, settings) from the primary Firebase project to the secondary backup project — and vice-versa. Run this whenever you've made significant changes (e.g. once a week) so the backup stays fresh. If the primary's free quota gets exhausted, you can flip window.FIREBASE_USE to 'secondary' in js/firebase-config.js and the live site instantly switches to the backup.
